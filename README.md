Propuesta de Sistema de Gestión de Tickets para Laboratorios de Cómputo
El presente documento tiene como objetivo proponer el desarrollo de un sistema de gestión de tickets enfocado en la administración de incidencias técnicas y solicitudes académicas en los laboratorios de cómputo. En la actualidad, el manejo de estos procesos se realiza de manera informal, lo que genera retrasos, falta de seguimiento y poca comunicación entre docentes y personal de soporte. Este sistema busca ofrecer una plataforma centralizada, moderna y eficiente que resuelva estas necesidades y aporte un valor diferencial frente a soluciones tradicionales.

El sistema estará orientado a dos perfiles de usuarios. Los profesores podrán registrar incidencias en los equipos de cómputo, como fallas de hardware o errores de software, además de realizar solicitudes académicas relacionadas con programas específicos que necesiten para sus clases. Cada profesor tendrá la posibilidad de consultar en tiempo real el estado de sus tickets y recibir notificaciones cuando estos sean atendidos, resueltos o rechazados. Por su parte, los administradores o técnicos de mantenimiento contarán con un panel de gestión donde visualizarán todas las incidencias y solicitudes creadas, las clasificarán según prioridad y darán respuesta a cada caso de manera organizada.

Para mejorar la comunicación entre profesores y técnicos, cada ticket contará con un chat interno integrado, donde ambas partes podrán conversar directamente sobre el problema o solicitud. Esta función no solo elimina la necesidad de depender de correos externos, sino que también genera un historial de comunicación accesible en cualquier momento. Complementando esta característica, el sistema enviará notificaciones en tiempo real mediante WebSockets, similares a las de aplicaciones de mensajería como WhatsApp o Slack. De esta manera, tanto profesores como técnicos estarán siempre informados de cambios de estado, respuestas y actualizaciones en los tickets.

El sistema también dispondrá de un módulo de notificaciones por correo electrónico, asegurando que ningún usuario pierda información relevante aunque no se encuentre conectado al sistema en ese momento. Asimismo, para los técnicos se incluirá una interfaz optimizada para dispositivos móviles, en la cual podrán visualizar de manera sencilla su lista de tareas pendientes, recibir notificaciones inmediatas y ser alertados automáticamente sobre aquellas tareas que lleven varios días sin resolverse. Esto permitirá un control más ágil del flujo de trabajo, incluso fuera de la oficina.

La solución tecnológica estará basada en un entorno web moderno, accesible desde navegadores y dispositivos móviles. Se desarrollará sobre un backend en PHP con el framework Laravel (o alternativamente en Node.js con Express), con un frontend en HTML, CSS y JavaScript utilizando librerías modernas como Bootstrap o Tailwind. La base de datos se gestionará en MySQL/MariaDB y se integrarán mecanismos de seguridad como contraseñas cifradas, gestión de roles y control de sesiones.

Los beneficios esperados de este sistema incluyen una reducción significativa en los tiempos de atención de incidencias, una comunicación más fluida y organizada entre docentes y técnicos, un registro histórico que permitirá generar reportes y planificar mantenimientos preventivos, y una mejora en la transparencia y eficiencia en la gestión de recursos tecnológicos de los laboratorios. Adicionalmente, la incorporación de funcionalidades como el chat interno, las notificaciones en tiempo real y la interfaz móvil para técnicos hacen que esta propuesta resalte frente a otras soluciones, posicionándola como una herramienta moderna y completa.

En conclusión, el sistema propuesto no solo resolverá la problemática actual de seguimiento de incidencias y solicitudes, sino que también ofrecerá un valor agregado con características avanzadas que optimizan la gestión, fortalecen la comunicación y garantizan una experiencia de usuario superior. Con ello, la institución contará con una plataforma que incrementará la productividad, reducirá los tiempos de respuesta y mejorará la calidad del soporte tecnológico en sus laboratorios de cómputo.
